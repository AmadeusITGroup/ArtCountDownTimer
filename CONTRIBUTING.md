# Contributing to ART Timer

Thank you for considering contributing to ART Timer! This document outlines the process for contributing to this project.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers understand your report:

- Use a clear and descriptive title
- Describe the exact steps to reproduce the issue
- Provide specific examples to demonstrate the steps
- Describe the behavior you observed and what you expected to see
- Include screenshots if applicable
- Include details about your environment (OS, browser, etc.)

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion:

- Use a clear and descriptive title
- Provide a step-by-step description of the suggested enhancement
- Provide specific examples to demonstrate the steps
- Describe the current behavior and explain the behavior you expected to see
- Explain why this enhancement would be useful to most ART Timer users

### Pull Requests

- Fill in the required template
- Do not include issue numbers in the PR title
- Follow the style guidelines
- Update documentation as needed
- Include tests that verify your changes
- End all files with a newline

## Development Environment Setup

1. Fork and clone the repository
2. Run `npm install` to install dependencies
3. Run `npm start` to start the application in development mode

## Style Guidelines

### JavaScript

- Use consistent indentation (2 spaces)
- Use semicolons
- Follow the existing code style in the project

### CSS

- Use consistent indentation (2 spaces)
- Use descriptive class names
- Follow the existing naming conventions

## Documentation

- Update the README.md with details of changes to the interface or functionality
- Update the documentation in the code as needed

## Testing

- Include tests for new features
- Run tests before submitting a pull request
- Make sure all tests pass

Thank you for contributing to ART Timer!
